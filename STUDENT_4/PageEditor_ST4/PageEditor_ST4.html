User Profile

Technical Description

  • The user profile page is built modern structural foundation using semantic HTML and advanced CSS layout techniques. The HTML document includes semantic tags like <main>, <header>, and <footer> to define a clear page hierarchy. Content is logically segmented into modular components using a repeatable <div class="card"> structure, which makes the layout clean and easy to manage.

  • The primary layout is powered by CSS Grid, creating a responsive two-column design.This structure is adaptive; a media query ensures the layout defaults to a single column on smaller screens, providing a mobile-first user experience. Within the grid, components utilize Flexbox (display: flex) for precise alignment of internal elements. This combination of Grid and Flexbox creates a highly efficient, maintainable, and responsive page structure.

    • Accessibility

    • Accessibility was addressed by using semantic HTML tags like <main> and <header> to provide a clear structure for screen readers. All images include descriptive alt attributes, and form inputs are correctly linked to <label> tags for clarity. The design maintains strong color contrast, ensuring readability for all users.

Content Page

Technical Description

  • This content page is structured using a clean, repeatable pattern for its sections. The primary layout technique is CSS Flexbox, used to create the side-by-side text and image arrangement in each content block.

  • A key feature is the use of a .reversed class, which applies flex-direction: row-reverse; to alternate the layout dynamically without altering the HTML structure. This creates a more engaging visual flow.

  • The page includes a "Dive In" navigation menu with anchor links for smooth intra-page jumps, enhanced by the scroll-behavior: smooth; property. A thoughtful scroll-margin-top is applied to the sections, ensuring content aligns perfectly in the viewport after a navigation click. Interactive elements feature subtle hover transitions, improving the user experience.

Accessibility

  • Accessibility is supported through a strong semantic structure using <main>, <section>, and hierarchical headings (<h1>, <h2>) for easy navigation. All images are provided with descriptive alt text, making the content accessible to screen readers. The page specifies its language as English and uses clear, descriptive text for all navigation links.

Sitemap

Technical Description

  • This page implements a unique and visually engaging sitemap using an inline SVG (Scalable Vector Graphic) instead of a traditional HTML list. The entire flowchart structure is drawn programmatically with SVG elements: <rect> shapes create the page nodes, <line> elements form the hierarchical connectors, and <text> provides the labels.

Accessibility

  • The layout is manually defined using a precise x and y coordinate system for each element. Interactivity is achieved by wrapping the SVG shapes in standard <a> tags, making the visual nodes clickable.

Challenges and Lessons Learned

Challenges

  • Aligning SVG elements in the Site Map page: Positioning elements precisely within an SVG-based sitemap was a challenging task. Unlike standard HTML layouts that rely on flexible containers like Flexbox or Grid, SVG requires manual placement using exact x and y coordinates. Ensuring that rectangles, lines, and text elements aligned correctly in both horizontal and vertical hierarchies took a lot of trial and error. I had to carefully calculate spacing, adjust for label lengths, and maintain symmetry across all levels of the sitemap.
  • Implementing smooth scrolling in the User Profile page: Another challenge was achieving smooth and intuitive scrolling behavior when navigating to internal sections. I wanted the transitions to feel polished and user-friendly, especially for anchor links like the “Back to Top” button. Initially, clicks on anchor tags resulted in abrupt jumps. To fix this, I applied the scroll-behavior: smooth; property on the HTML tag, but it took experimentation to ensure compatibility across browsers. Additionally, I had to manage scroll-margin-top to offset fixed headers, ensuring that targeted sections were not hidden behind the nav bar after jumping.
  • Designing the calendar using tables in HTML: Creating a calendar layout using only HTML tables was unexpectedly complex. I needed to structure it in a way that visually resembled a real calendar, with correct alignment for weekdays and proper date placement.

Lessons Learned

  • Learned how to work with SVG graphics to create interactive and visually structured diagrams using manual positioning with x and y coordinates.
  • Learned to troubleshoot layout issues and manually align visual elements in non-flexible environments like SVG, which enhanced my attention to design precision.
  • Improved understanding of responsive design techniques using CSS Grid and Flexbox, and how to combine them effectively for complex layouts.
  • Strengthened knowledge of web accessibility principles, including semantic HTML, form labeling, and the use of descriptive alt text for screen readers.

Compliance

Ensuring compliance with web standards and accessibility guidelines is essential for a user-friendly and legally sound website. The following principles were adhered to:

  • HTML & CSS Standards: All pages follow HTML5 and CSS3 best practices, ensuring proper structure and styling.
  • Accessibility Compliance: Implemented semantic HTML and high contrast color schemes for better readability.
  • Responsive Design: Used flexible layouts and media queries to provide a seamless experience across different screen sizes.
  • Privacy & Security: Ensured form submissions do not store sensitive user data, complying with basic GDPR principles.
  • SEO Best Practices: Structured data, and descriptive alt text were used to improve search engine visibility.
  • JANET Compliance: All pages adhere to JANET guidelines by avoiding harmful or inappropriate content, using only properly credited or royalty-free resources, and ensuring user privacy by not collecting or storing any personal data.

By following these principles, the website ensures compliance with modern web regulations and enhances user experience.
